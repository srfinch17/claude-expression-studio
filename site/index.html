The bigger idea

One message. Any screen.

Under the panel is a tiny protocol. Claude sends one semantic message (an intent, maybe some data) and any display you've connected renders it its own way. The 8×8 is just the first renderer.

// Claude says it once:
presence.set({ intent: "working", headline: "Running tests", data: { progress: 0.6 } })

Get it

No board required. Fully open.

Install it and Claude renders its expressions on a virtual panel right in your browser. A physical Waveshare ESP32-S3-Matrix (a ~$15 board, 8×8 WS2812B) is an optional upgrade you can add anytime. Three steps:

01 · Install

Add it to Claude

Double-click the .mcpb extension from this repo for Claude Desktop, or run npm run setup for Claude Code. No JSON, no config, no hardware.

02 · Say hi

Watch it come alive

Claude starts expressing on its own: a spinner when it's busy, a check when it's done, a tap when it needs you. It renders on the in-browser virtual board and the draggable desktop mini board. See a loop you love? Tell Claude to pin it (or click it in the board's pin strip) and the auto-rotation holds off until you say otherwise.

03 · Add a board

Optional: put it on your desk

Want a glowing thing on your desk? Flash a ~$15 panel from the board firmware repo (flash.bat / flash.sh, one merged image). The same tools start mirroring to it at esp32matrix.local. Nothing else changes.

Where it stands: the Studio, the Claude integration, and the board-free virtual panel all work; the firmware and its web control panel are hardware-verified. Tagged releases and a one-click browser flasher are on the way. Full instructions and source are on GitHub.
